# 障害者総合支援法 クイズ学習ツール

障害者総合支援法を学習するための自分専用クイズ学習ツールです。

## 技術スタック

| 項目 | 採用技術 |
|---|---|
| フレームワーク | Next.js 15（App Router） |
| 言語 | TypeScript |
| スタイル | Tailwind CSS |
| データベース | Neon（PostgreSQL） |
| DBクライアント | @neondatabase/serverless |
| デプロイ | Vercel |

---

## 何を保存できるようにしたか

- **テーマ**（補装具・支給決定・障害支援区分など）の登録・削除
- **問題集**（テーマごと・初級/中級/上級のレベル別）の管理
- **問題**（4択問題・正解・解説）の手動登録
- **解答履歴**（正解/不正解・解答日時）の自動保存

---

## どのデータをNeonに保存したか

| テーブル | 保存内容 |
|---|---|
| `themes` | テーマ名・作成日時 |
| `problem_sets` | 問題集（テーマID・レベル・作成日時） |
| `questions` | 問題文・選択肢A〜D・正解・解説 |
| `answer_history` | 解答履歴（問題ID・正誤・解答日時） |

苦手問題は専用テーブルを持たず、`answer_history` から `is_correct = false` の問題を集計するビュー（`weak_questions`）で動的に取得しています。

---

## なぜNeonを選んだか

- **Vercelとの統合が簡単**：VercelのStorageタブから直接追加でき、環境変数も自動連携される
- **サーバーレス対応**：`@neondatabase/serverless` パッケージがEdge/Serverless環境に最適化されている
- **PostgreSQL互換**：ビューやCASCADE削除など標準SQLが使える
- **無料枠あり**：個人学習ツールとして十分な容量

---

## どのデータはリポジトリ側に残したか

- `db/schema.sql`：テーブル定義・ビュー・サンプルデータ（SQLファイルとして管理）
- `.env.example`：環境変数のテンプレート（実際の接続文字列は含まない）

`.env.local`（実際のNeon接続文字列）は `.gitignore` で除外しています。

---

## 工夫したポイント

- **苦手問題の動的抽出**：専用テーブルを作らず、`answer_history` から `is_correct = false` を集計するビューを使用。常に最新状態を反映する
- **カスケード削除**：テーマを削除すると問題集→問題→解答履歴まで連動削除される（`ON DELETE CASCADE`）
- **タブ切替クイズ**：ペイン②で「通常モード」と「苦手問題モード」をタブで切り替えられる
- **リアルタイム更新**：解答するたびにペイン④の統計（正答率・苦手問題数）が更新される
- **重複チェック**：同名テーマの二重登録を `UNIQUE` 制約で防止

---

## 苦戦したポイント

- **`.env.local` の保存問題**：Cursorエディタで編集した内容がディスクに保存されておらず（0バイト）、環境変数が読み込まれなかった。Writeツールで直接書き込むことで解決
- **フォルダ名のスペース問題**：`create-next-app` がnpm命名規則でフォルダ名（`20260613 quiz`）を拒否したため、手動でプロジェクトファイルを作成
- **複数サーバープロセスの競合**：開発中に複数の `next dev` が起動してポートが競合した

---

## 今後追加したいAI生成機能

テーマ名を入力するだけで、OpenAI APIを使って問題を自動生成する機能を追加予定。

**生成内容：**
- 初級10問・中級10問・上級10問（合計30問）
- 問題形式：4択問題
- 各問題に問題文・選択肢A〜D・正解・解説をJSON形式で生成

**実装イメージ：**
```
入力：「補装具」
  ↓
OpenAI API（GPT-4o）がJSON生成
  ↓
問題集・問題テーブルに一括INSERT
  ↓
すぐにクイズで解ける状態になる
```
